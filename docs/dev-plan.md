# 磁律禁区 — 开发计划

## 阶段一：基础框架

### 1.1 项目骨架
- [ ] 确定 Godot 版本、项目结构
- [ ] 创建主场景、GameManager 单例
- [ ] 横屏适配（Viewport 设置、安全的 CanvasLayer 缩放方案）
- [ ] 场景切换管理（主菜单 → 游戏 → 结算）

### 1.2 自定义物理系统
- [ ] 实现轻量 Vector2 运动模型（位置 + 速度 + 加速度）
- [ ] 实现 AABB 矩形碰撞检测
- [ ] 实现简单的碰撞响应（仅垂直/水平方向分离）
- [ ] 暴露每帧手动步进接口（`_process(delta)` 驱动），不依赖 `_physics_process`

---

## 阶段二：场景卷动系统

### 2.1 世界坐标与视口
- [ ] 确定世界坐标系：角色 X 固定在屏幕中央偏右，场景沿 X 轴负方向后退
- [ ] 世界坐标 → 屏幕坐标 转换工具函数

### 2.2 双跑道架构
- [ ] 定义上跑道 Y 坐标（地板线）和下跑道 Y 坐标
- [ ] 天花板线相对于各自跑道顶部的偏移量

### 2.3 卷动控制器
- [ ] ScrollManager：管理全局卷动速度
- [ ] 支持上下跑道**独立速度**
- [ ] 运行时动态调速接口（供特殊关卡使用）

---

## 阶段三：角色系统

### 3.1 角色基础
- [ ] Character 场景/类：矩形碰撞体、跑道归属、磁性属性
- [ ] 默认状态：在地板线上站立/奔跑
- [ ] 垂直移动通道：地板 ↔ 天花板（受磁力驱动）
- [ ] 天花板状态：在天花板线上倒置行走

### 3.2 角色移动
- [ ] 水平位置维持逻辑（随场景后退保持 X 不变）
- [ ] 垂直移动：无磁力时在所属地板/天花板行走，有磁力时向目标面移动
- [ ] 阻挡时与场景墙碰撞停止水平跟随

### 3.3 角色交换
- [ ] SwapManager / Swap 按钮输入
- [ ] 交换两个角色的跑道归属
- [ ] 交换过程动画（可选：直接瞬移，后续补过渡动画）

---

## 阶段四：磁力系统

### 4.1 场景磁铁
- [ ] Magnet 场景/类：位置、长度、极性（N/S）、所属面（地板/天花板）
- [ ] 磁铁仅产生垂直方向的平行力场，力场范围 = 磁铁长度（AABB 矩形）
- [ ] 可视化调试：绘制力场矩形

### 4.2 磁力交互
- [ ] 每帧检测每个角色是否落入任意磁铁的力场矩形内
- [ ] 如果角色磁性激活且极性匹配磁铁 → 施加吸引力（向上/向下）
- [ ] 到达目标面（天花板/地板）后停止施加力
- [ ] 离开力场范围后磁力消失，角色停留在当前所在面

### 4.3 输入绑定
- [ ] 角色 A 磁力按钮（按下持续触发，松开停止）
- [ ] 角色 B 磁力按钮（同上）
- [ ] 区分持续按住与单击逻辑

---

## 阶段五：障碍与碰撞

### 5.1 墙
- [ ] Wall 类：矩形碰撞体，属于某个跑道
- [ ] 角色碰到墙 → 角色被阻挡，无法跟随场景后退
- [ ] 角色与墙的碰撞方向仅为水平方向

### 5.2 碰撞系统
- [ ] 每帧收集所有角色碰撞体与所有场景障碍碰撞体
- [ ] AABB 检测 → 水平分离 → 更新角色 X 位置

---

## 阶段六：地图生成

### 6.1 区块定义
- [ ] 设计 LevelChunk 基类/场景：包含该段的地板、天花板、磁铁、墙的配置
- [ ] 区块固定宽度（如 640px）
- [ ] 设计 3-5 种基础区块模板（普通、磁铁区、墙区、混合区）

### 6.2 生成与销毁
- [ ] ChunkSpawner：在屏幕右侧按一定间隔生成新区块
- [ ] 当前区块完全移出屏幕左侧后销毁（queue_free）
- [ ] 维护活跃区块列表，控制上限（如最多 5-6 块）
- [ ] 难度递进：随距离增加，磁铁变短、墙更密集

### 6.3 双跑道独立生成
- [ ] 上跑道和下跑道可分别配置各自的区块模板
- [ ] 支持上下跑道不同步生成（为独立速度做铺垫）

---

## 阶段七：尖刺与回中

### 7.1 尖刺
- [ ] Spike 场景：始终位于屏幕左侧固定偏移位置
- [ ] 尖刺以全局速度向右追赶（与场景后退速度一致）
- [ ] 任意角色 X 位置 ≤ 尖刺 X 位置 → 游戏结束

### 7.2 回中机制
- [ ] 角色无阻挡时，每帧以一定速率向右回移到默认 X 位置
- [ ] 回中速度可配置
- [ ] 回中仅在角色未被墙阻挡时生效
