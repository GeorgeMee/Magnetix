# 磁律禁区 — 游戏设计文档

## 1. 概述

- **游戏名称**：磁律禁区
- **平台**：移动端（手机/平板）
- **画面方向**：横屏
- **游戏类型**：2D 本地单机跑酷
- **玩法原型**：《一个都不能死》多跑道平台卷轴游戏
- **核心机制**：磁力吸附

---

## 2. 核心玩法

### 2.1 双跑道双角色

游戏包含上下两条平行跑道，每条跑道上各有一个角色：

| 跑道 | 角色 | 磁性 |
|------|------|------|
| 上跑道 | 角色A | 某种磁极 |
| 下跑道 | 角色B | 相反/不同磁极 |

### 2.2 磁力机制

- **触发方式**：玩家**点击按键**触发对应角色的磁性，**非**常驻状态。玩家松开按键即可关闭。
- **持续触发**：在磁场区域内，玩家需持续按住按键以保持磁性，确保角色能完整走过天花板路段。
- **磁场区域**：
  - 场景中分布着固定在**地板**或**天花板**上的磁铁。
  - 磁铁产生的是**矩形平行力场**——力场长度由磁铁本身的长度决定，而非圆形半径。
  - 磁铁只提供**垂直方向的力**（向上或向下），**不提供水平方向的力**。
  - 角色进入磁铁的矩形力场范围后，根据磁性（吸引/排斥）获得向上或向下的力。
- **引力表现**（向上力）：角色被吸附到天花板上行走，从而**避开地面障碍物**。
- **斥力表现**（向下力）：角色被压向地板。

### 2.3 角色交换

- 玩家可通过一个**交换操作**（例如点击交换按钮）互换两个角色的位置：
  - 上跑道的角色换到下跑道，下跑道的角色换到上跑道。
  - 交换后，角色可利用对应位置磁铁的磁性进行交互。

### 2.4 跑道独立控制

由于采用"玩家原地、场景后退"的模式，两条跑道的**卷动速度可以独立控制**，用于实现：

- **危险跑道**：一条跑道速度加倍，障碍更密集
- **安全跑道**：一条跑道速度减缓，给玩家喘息空间
- **奖励关卡**：双跑道速度降低，障碍减少
- **特殊关卡**：一条跑道出现特殊磁铁分布或特殊障碍物

## 3. 场景系统

### 3.1 卷动模式

- **玩家在原地不动**（位于屏幕中央偏右），**场景向左后退**。
- 这样做的优势：
  - 方便判断地图的生成与销毁逻辑。
  - 可以分别控制两条跑道的卷动速度。
  - 玩家始终在固定位置，操作反馈直观。

### 3.2 地图生成与销毁

- 关卡为 **Endless（无尽）** 模式。
- 地图采用**区块生成**策略：
  - 在屏幕右侧不断生成新的场景区块。
  - 区块移动到屏幕左侧一定距离后销毁。
  - 控制场景中同时存在的区块数量，避免过多导致性能卡顿。

### 3.3 危险机制：尖刺追赶

- 屏幕**左侧**有一个尖刺（或致命边界）以固定速度向右追赶玩家。
- 尖刺是即死判定：被追上则游戏结束。
- 尖刺的存在确保玩家必须不断前进（场景不断后退）。

### 3.4 障碍物

- **墙**：场景中的障碍物，角色碰到墙会被**挡住**。
- 被墙挡住 → 角色相对场景停止 → 场景继续后退 → 角色离左侧尖刺更近 → 增加紧迫感。
- 当前阶段仅设计**墙**这一种障碍物，后续可扩展。

### 3.5 回中机制

- 角色被墙挡住后会偏离默认位置（偏向左侧）。
- 需要一个**缓慢回到屏幕中心位置**的机制：
  - 例如：角色没有被阻挡时，每帧以固定速度向右回移，直至回到默认 Y 轴位置。

## 4. 操作设计

| 操作 | 效果 |
|------|------|
| 点击角色A磁性按钮 | 触发/持续触发角色A的磁性 |
| 点击角色B磁性按钮 | 触发/持续触发角色B的磁性 |
| 点击交换按钮 | 交换两个角色的跑道位置 |

> 操作按键保持按下时磁性持续生效，松开即停止。在磁场区域内需持续按住以确保稳定吸附。

---

## 5. 物理系统

- 采用**自定义物理系统**，不使用 Godot 自带的物理引擎。
- 原因：
  - 游戏只需要有限的方向力（仅垂直方向的磁力），无需完整物理模拟。
  - 自定义物理更轻量，便于精确控制角色移动和磁力行为。
  - 双跑道独立速度控制需要自定义运动逻辑。

### 5.1 物理规则

- **角色水平移动**：由场景卷动速度决定（角色本身不水平移动，场景后退）。
- **角色垂直移动**：
  - 默认在地板（跑道）上行走。
  - 触发磁性时，根据与场景磁铁的磁性关系，获得向天花板或向地板的力。
  - 到达天花板后在天花板上行走，可躲避地面障碍。
- **磁铁作用范围**：磁铁产生矩形平行力场，力场长度 = 磁铁长度。角色只有进入矩形力场范围内才能产生磁力交互。
- **碰撞**：自定义矩形碰撞检测处理角色与墙的碰撞。

---

## 6. 视觉与表现

- 2D 平面横版表现。
- 双跑道上下排列。
- 天花板和地板各有磁铁分布（可用不同颜色/标记区分 N/S 极）。
- 尖刺位于屏幕左侧，始终可见。
- 角色与磁铁交互时应有视觉反馈（磁力线、光效等）。
